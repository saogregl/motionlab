# Development Workflow

## Default Flow

1. Create or refine an issue with a linked brief.
2. Read `AGENTS.md`, the architecture index, and the relevant subsystem docs.
3. Use the relevant skill when the task touches architecture, protocol, sensors, results, or docs.
4. Implement the change in a narrow slice.
5. Update docs, ADRs, and tests in the same change when required.
6. Run local checks before opening a PR.

## Required Inputs For Meaningful Work

- goal and non-goals
- affected modules
- architecture constraints
- tests required
- docs to update
